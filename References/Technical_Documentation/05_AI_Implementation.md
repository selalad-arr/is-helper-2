# 05 AI Implementation - IS Helper

## บทบรรยาย: ปัญญาประดิษฐ์ในฐานะพันธมิตรทางการศึกษา
หัวใจของ IS Helper คือการนำ AI มาใช้ในเชิงสร้างสรรค์ ไม่ใช่เพื่อโกงงาน แต่เพื่อเป็น "ผู้ช่วยเรียบเรียง" เอกสารนี้จะบรรยายลำดับการทำงานของ Gemini AI ว่าระบบมีการควบคุมขอบเขตของเนื้อหาอย่างไรเพื่อให้มั่นใจว่าผลลัพธ์ที่ได้นั้นมีความเหมาะสมกับมาตรฐานวิชาการของไทย


## หัวใจด้าน AI: Gemini API
แอปพลิเคชันเลือกใช้ **Google Gemini (รุ่น 1.5 Flash)** เป็นเครื่องมือหลักเนื่องจากความเร็วและความสามารถในการเข้าใจภาษาไทยที่โดดเด่น

## วิธีการทำงาน (Flow)
1.  **Prompt Engineering:** เมื่อนักเรียนกดขอความช่วยเหลือ ระบบจะไม่ส่งแค่คำขอสั้นๆ แต่จะส่ง "Context" ไปด้วย เช่น "คุณคือผู้เชี่ยวชาญด้านงานวิจัยวิชา IS ของไทย และนี่คือข้อมูลพื้นฐานที่นักเรียนเตรียมไว้..."
2.  **Constraint Setting:** มีการกำหนดให้ AI ไม่เขียนเรื่องที่เป็นอันตราย หรือเรื่องที่ไม่เกี่ยวข้องกับงานวิจัยวิชาการ
3.  **Output Formatting:** กำหนดให้ AI ส่งข้อมูลกลับมาในรูปแบบที่แอปสามารถนำไปแสดงผลต่อได้ทันที (Plain text หรือ Markdown)

## การจัดการระบบโควตา (Quota Management)
เพื่อให้ระบบอยู่รอดได้โดยไม่มีค่าใช้จ่ายเซิร์ฟเวอร์บานปลาย:
- **Free Users:** จำกัดการเรียกใช้ AI ผ่านระบบหลัก (3 ครั้ง/วัน) เพื่อประหยัด API Token
- **Self-Key Users:** ระบบจะสลับไปใช้ API Key ที่นักเรียนเก็บไว้ใน LocalStorage ทำให้ไม่เสียโควตาของระบบหลัก
- **Premium Users:** อนุญาตให้ใช้ระบบหลักได้ไม่จำกัด โดยระบบจะทำการ Priority คำขอของกลุ่มนี้ก่อน

## แชทบอทอัจฉริยะ (Floating Assistant)
ใช้ `global_assistant` context ซึ่งถูกสอนให้รู้เรื่องระเบียบการทำโครงงาน IS ในประเทศไทยโดยเฉพาะ ทำให้คำแนะนำมีความน่าเชื่อถือและเหมาะสมกับบริบทนักเรียนไทย
